В gitlab описание таксов очень похожее на github 
Зеленым цветом в прогрессе выполнения обозначены команды которые мы написаил в gitlab-ci.yml файле
Установку node можно вынести в бефоре скрипт, для того чтобы установка произошла до выполнения остальных команд
Если запускать один stage для нескольких jobs они будут выполняться параллельно - если нужно больше контроля то можно рассмотреть DAG, чтобы указать зависимость между jobs
Allow failure - или что-то похожее, настройка, которая позволяет даже в случае падения job продолжить процесс CI
